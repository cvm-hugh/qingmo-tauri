# 轻墨 2.0

轻墨 2.0 是基于原始轻墨 1.0.0 版本整理和继续优化的本地 Markdown 编辑器版本。它保留了轻墨原来的双栏写作方式，并加入了更接近文档编辑器的预览区编辑能力。

## 当前功能

- 支持打开、编辑、保存 `.md`、`.markdown`、`.txt` 文件。
- 支持多标签编辑。
- 左侧为 Markdown 源码编辑区，右侧为实时预览区。
- 右侧预览区默认可直接编辑文字。
- 左侧源码修改后，右侧预览会实时更新。
- 右侧文字编辑后，会同步回左侧 Markdown。
- 支持撤销和重做。
- 支持有序列表、无序列表和任务列表。
- 支持任务列表语法：`- [ ] 任务`、`- [x] 已完成任务`。
- 支持列表缩进层级，最多 4 层。
- 左侧编辑区支持 `Tab` 降低列表层级，`Shift+Tab` 提高列表层级。
- 左侧有序列表在调整层级后会重新编号。
- 支持标准 Markdown 引用语法：`> 引用内容`。
- 引用内容支持多行，直到空行结束。
- 引用内支持列表和任务列表。
- 支持表格语法预览。
- 支持基础样式：加粗、斜体、下划线、标题、行内代码、高亮、引用、表格。
- 支持高亮文字，语法为 `::文本::`。
- 保留富文本粘贴模式，可用于整篇粘贴网页、文档或其他富文本内容并转换为 Markdown。
- 支持导出 PDF。
- 支持导出 Word 文档。

## 主要特色

- 保留“源码 + 预览”的双栏写作方式。
- 右侧预览区可以直接修改文字，适合快速校对和轻量编辑。
- 富文本粘贴模式仍然保留，适合把外部格式化内容整体导入为 Markdown。
- 列表、任务列表和引用比原始版本更接近常见 Markdown 编辑器的使用习惯。
- 所有优化都基于可复现的原始轻墨版本源码，不影响已备份的原始 app。

## 已知限制

- 不支持图片插入、图片嵌入或图片预览。
- Markdown 文件本身不会保存图片内容；图片通常需要额外的资源文件夹管理。本版本暂不实现该功能。
- 右侧预览区可以编辑文字，但复杂结构编辑仍建议在左侧 Markdown 源码区完成。
- 右侧预览区输入 `- `、`1. ` 不自动转换为列表；建议通过左侧源码区编辑列表结构。
- 右侧预览区的列表层级编辑不是主要工作流；列表格式建议以左侧源码区为准。
- 高亮语法只支持同一行内的文字；跨行高亮需要在每一行重新输入 `::` 边界符号。
- 复杂 HTML 富文本粘贴转换为 Markdown 时，可能无法完全保留原始排版。
- PDF 和 Word 导出的版式适合普通文档，不等同于专业排版软件。
- 当前版本仍是本地自用版本，未做公开分发、公证或通用安装包适配。

## 推荐使用方式

- 写作和结构调整：优先使用左侧 Markdown 源码区。
- 校对和少量文字修改：可以直接在右侧预览区完成。
- 外部内容导入：使用富文本粘贴模式。
- 列表、任务、引用等结构化内容：建议在左侧源码区编辑，右侧用于预览确认。
- 图片内容：建议暂时放在其他文档或资源管理工具中，不作为轻墨 2.0 的 Markdown 内容管理。

## 版本定位

轻墨 2.0 的目标是成为一个轻量、本地、适合 Markdown 写作和整理的编辑工具。它强调快速写作、实时预览、富文本导入和基础文档导出，不追求完整替代 Typora、Word、Notion 或专业笔记库软件。
